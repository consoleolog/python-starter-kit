> **Commit은 꼭 필요한 최소한의 단위로, 충분한 설명과 함께!**

---

## 1. Commit 원칙

> **Layer 기반의 "최소화" 개발**: 작은 기능, 작은 변경, 작은 책임

- Commit에 대응되는 **GitHub Issue를 먼저 생성**
- 기능 반영 시 test code를 **반드시 함께** commit
- **최대한 작은 단위**로 commit, 최대한 100줄 이하로 개발 (100줄 이하 **테스트, 주석 제외**, 넘어갈 것 같으면 커밋 분리)

---

## 2. Commit 메시지 형식

```
제목: 1줄로 내용 요약. 다른 commit과 동일한 제목이 되지 않도록 구분하여 작성 ← Header (필수)

내용: 수정한 내용에 대한 자세한 설명.          ← Body (선택)
주의사항, 우려되는 내용들도 함께 기재 권장.
미완료 사항은 GitHub Issue를 생성하여 명시.

close #<이슈번호>                             ← Footer (필수)
```

---

## Header (헤더)

한 줄로 변경 사항을 요약합니다. **50자 이내**로 작성합니다.

```
<타입>: <제목>
```

### 타입 목록

| 타입         | 설명                           |
|------------|------------------------------|
| `feat`     | 새로운 기능 추가                    |
| `fix`      | 버그 수정                        |
| `docs`     | 문서 수정                        |
| `style`    | 코드 포맷팅, 세미콜론 누락 등 (로직 변경 없음) |
| `refactor` | 코드 리팩토링 (기능 변경 없음)           |
| `test`     | 테스트 코드 추가 또는 수정              |
| `chore`    | 빌드 설정, 패키지 관리 등 기타 변경        |
| `perf`     | 성능 개선                        |
| `ci`       | CI/CD 설정 변경                  |
| `revert`   | 이전 커밋 되돌리기                   |

---

## Body (본문)

**무엇을**, **왜** 변경했는지 설명합니다. **한 줄에 72자 이내**로 작성합니다.

- 헤더와 한 줄 빈 줄로 구분
- 여러 항목은 `-` bullet point로 나열

---

## Footer (꼬리말)

```
<키워드> #<이슈번호>
```

PR 머지 시 해당 Issue가 자동으로 닫힙니다. 작업 성격에 따라 아래 키워드를 선택합니다.

| 키워드                          | 사용 상황                          |
|---------------------------------|------------------------------------|
| `close` / `closes` / `closed`   | 기능 추가, 문서, 설정 등 일반 작업  |
| `fix` / `fixes` / `fixed`       | 버그 수정 (`fix` 타입 커밋)         |
| `resolve` / `resolves` / `resolved` | 논의·개선 요청·질문 등 해결      |

> 세 키워드 모두 동일하게 Issue를 닫으며, 의미 전달 목적으로 구분합니다.

---

**작성 예시:**

```
fix: 주소록 정렬할 때 여러 자리 숫자에 대응할 수 있도록 comparator 변경하여 bug 대응

기대값: 홍길동1, 홍길동2, 홍길동11의 순서로 정렬
오류: 홍길동1, 홍길동11, 홍길동2의 순서로 정렬

기본 comparator로 숫자가 올 경우 int로 변경하여 대소 비교하도록 수정하여 대응함.
* contact_comparator.py : 위 방법대로 정의한 custom comparator 구현
성능 검증은 추후 예정

fix #12
```
